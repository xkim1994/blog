JavaScript设计模式之命令模式

发表于 2020-04-25 更新于 2021-07-14 分类于 DesignPattern

命令模式

命令模式旨在将方法调用，请求或操作封装到一个对象中，并使我们能够对参数调用进行参数化和传递，这可以由我们自行决定执行。另外，它使我们能够将调用动作的对象与实现它们的对象分离开来，从而在交换具体类（对象）时具有更大的整体灵活性。

具体的类最好用基于类的编程语言来解释，并且与抽象类的思想有关。一个抽象类定义了一个接口，但并不一定对所有的它的成员函数提供实现。它充当其他派生对象的基类。实现缺少功能的派生类称为具体类。

Command 模式背后的总体思想是，它为我们提供了一种方法，可以将发出命令的职责与任何执行命令分开，而将这种职责委派给不同的对象。

在实现方面，简单的命令对象将动作和希望调用该动作的对象绑定在一起。它们始终包含执行操作（例如 run()或 execute()）。具有相同接口的所有 Command 对象都可以根据需要轻松交换，这被认为是该模式的更大好处之一。
